Interrupts: http://codebase64.org/doku.php?id=base:introduction_to_raster_irqs Space view is in standard bitmap mode (2 colours, 320x200 pixels) Comprises a bitmap, plus colour info in screen RAM Screen RAM has one byte for each character block This defines the two-colour palette in that block %0 in bitmap -> pixel in background colour, i.e. bottom nibble of SRAM %1 in bitmap -> pixel in foreground colour, i.e. top nibble of SRAM Dashboard is in multicolour bitmap mode (4 colours, 160x200 pixels) Comprises a bitmap, plus colour info in screen RAM and colour RAM Screen RAM and colour RAM both have one byte for each character block Together they define the four-colour palette in that block %00 in bitmap -> pixel in background colour, i.e. colour in $D021 %01 in bitmap -> pixel in colour in top nibble of screen RAM %10 in bitmap -> pixel in colour in bottom nibble of screen RAM %11 in bitmap -> pixel in colour in bottom nibble of colour RAM (this is sdump/cdump = screen and colour RAM colour info for dashboard) Split screen is implemented in COMIRQ1 interrupt routine Good explanations of screens modes: https://sta.c64.org/cbm64disp.html https://sta.c64.org/cbm64col.html
