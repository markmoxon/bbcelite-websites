How the game adds objects to the tile landscape

The deep dive on generating the landscape explains how The Sentinal creates 10,000 different landscapes, using the landscape seed generator and a collection of smoothing and scaling algorithms to convert 1024 random numbers into a tiled landscape for the player to explore.

But before the game can commence, we need to add a few things to the landscape, namely the player, the enemies and a collection of trees. There are two places where this is done:

  • If we are about to play a new landscape, then the PreviewLandscape takes over from the GenerateLandscape routine once the latter has generated the tile data. PreviewLandscape then adds enemies, displays the preview screen, and then adds the player and the trees.
  • If the player has just beaten the Sentinel, then we need to display the secret code for the next landscape that's just been unlocked. To do that we call the GetNextLandscape routine to generate the new landscape and add all the objects but without previewing the landscape, as this is how we get the secret code (see the deep dive on the landscape secret code for details).

The rest of this article explores how the enemies, player and trees are added to the landscape, using more landscape seeds and more algorithms.
